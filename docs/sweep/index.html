Methodology

Dial grid (48 configurations)

Three dials of the COLA configuration space (Definition 2, §3 of the paper) are swept; the remaining four are held at Classic-COLA defaults.

DialValuesSemantics
E — eligibility set {14, 22, 16-tiered} 14 = Classic pool (non-playoff teams). 22 = Simple / Capped pool (14 non-playoff + 8 first-round losers). 16-tiered = NBA 3-2-1 proposal pool.
C — stockpile cap {∞, 100, 150, 200} ∞ = no cap (Classic / Simple). 150 = Highley Capped-COLA Substack default. 100 / 200 are tighter / looser caps.
S — carry-over scope {single-season, unbounded, bounded-30yr, reset-on-championship} single-season = no cross-year memory (3-2-1 baseline). unbounded = Classic multi-year. bounded-30yr = 30-year rolling window. reset-on-championship = clear index when a team wins a title.
Fixed at Classic defaults Δ = 1000 (increment) · ρ = playoff-success-step (diminishment) · W = uniform (lottery weighting) · T = coin-flip (tiebreak)

Four objectives reported per configuration

Pareto-optimality is computed under 3-objective universal dominance on max-yrs-CF, manipulation-gain, and rank-1-to-5 spread. per_series_cost_typical is reported as a 4th disclosure column (n/a for uncapped) but is not used as a dominance dimension.

Simulation model

Real ZenGM lottery + synthesized regular-season game outcomes (strength-weighted wins, single-elim playoff bracket). Persistent team-strength Markov:

$s_{t+1} = \mathrm{clip}\bigl(\rho\, s_t + (1 - \rho)\,\mu + \alpha\,\pi(p_t) + \varepsilon_t,\ 0,\ 1\bigr)$

with $\rho = 0.9$, $\mu = 0.5$, $\alpha = 0.15$, $\sigma = 0.05$, initial strength $\sim \mathrm{Uniform}[0.3,\, 0.7]$, and pick-value $\pi(p) = \max(0, (16 - p) / 15)$. Calibration is parametric (smoke-tuned for plausibility), not fit to historical team-strength trajectories.

Replicates & sensitivity

50 replicates per configuration at the headline. Sensitivity at N=30 and N=100 on a 9-config subset (5 Pareto + 5 named, deduplicated). No flagged drift — all sensitivity-subset configs hold below 10% relative shift in mean(max_yrs_CF) between N=30 and N=100.

Cap-cancels-out: empirical confirmation

Manipulation gain is invariant under C for capped configurations

Corollary 2.1 of the paper predicts that the capped manipulation-gain bound $\Delta p_{\mathrm{capped}} \leq 0.3\, /\, |E|$ depends only on the eligibility size $|E|$, not on the cap value $C$. The headline sweep confirms this analytically: every capped configuration at the same E sits at exactly the same manipulation gain across C ∈ {100, 150, 200}.

E C = 100 C = 150 C = 200 $0.3 / |E|$

16-tiered 1.8750% 1.8750% 1.8750% 1.875%

Each cell is the manipulation-gain median for the four S-variants at that (E, C) pair — all four collapse to the same closed-form value. The 16-tiered approximation uses $|E| = 16$ pool seats.

Assumptions worth flagging

These are the 8 assumptions worth flagging at signoff. Each materially shapes the interpretation of the Pareto frontier above. The comprehensive list lives in scripts/basketball_gm_sweep/ASSUMPTIONS.md; the items below are the distilled subset.

ZenGM pool size left at 22, behavior matched to 14

ZenGM's default COLA pool is 22 teams (14 non-playoff + 8 first-round losers), not 14. We recover the 14-team Classic pool by zeroing the COLA index of R1 losers before the lottery draw, leaving ZenGM's source unmodified.

Why it matters: lottery weights are correct, but the pool-size constant inside ZenGM still reads 22 for any future diagnostic. Flag if you'd prefer a per-config source patch.

Dial-space reduced to E × C × S = 48

$\Delta$, $\rho$, $W$, $T$ are held at Classic defaults. $\Delta$ scales the bound linearly (already in Theorem 1); $\rho$, $W$, $T$ variations require ZenGM-source modifications with no published-policy anchor to compare against.

Why it matters: if you want any of these in the headline grid (e.g., $\rho$ = playoff-success-linear vs. step), we should add it before locking.

Lottery is real ZenGM; game outcomes synthesized; strength is Markov

ZenGM's full game engine requires a browser-side createLeague() invocation impractical to port to Node. We synthesize regular-season outcomes and evolve team strength via:

strength_{t+1} = clip(rho * strength_t + (1 - rho) * mu + alpha * pick_value(pick_t) + eps_t, 0, 1)

Tuned: $\rho = 0.9$, $\mu = 0.5$, $\alpha = 0.15$, $\sigma = 0.05$. Initial strength $\sim$ Uniform[0.3, 0.7]. Pick-value monotone-decreasing: pick 1 = full $+\alpha$, pick 15 = 0, picks 16+ = 0.

Why it matters: synthesis bypasses contracts / trades / injuries (none of which the dial space controls) but preserves the feedback loop the primary objective measures. Calibration is parametric, not fit to historical NBA trajectories.

30-season horizon, 50 replicates at headline

30 years lets the bounded-30yr scope manifest. Sensitivity at N = 30 / 100 on 9 representative configs verifies the Pareto frontier is not a Monte Carlo artefact.

Why it matters: total headline cost is 48 × 50 = 2,400 simulated seasons, ~3s per replicate at the 30-season horizon.

Forward-simulated, not historical

This testbed answers: "what does each dial setting look like under a synthetic but ZenGM-calibrated 30-team league?" — distinct from the COLA Explorer backtest over NBA 1999–2025.

Why it matters: the paper needs both: backtest for historical anchoring (Capped@150 calibration; the "Process" replication); forward-sim for the Pareto frontier across counterfactual dial settings.

Lottery-draw nondeterminism propagates into strength

ZenGM's lottery draw uses an unseeded Math.random (the driver's mulberry32 only seeds the synthetic season / bracket sim). Under the Markov model the draft pick feeds next season's strength, so two single-replicate runs with identical mulberry32 seeds can produce materially different max-yrs-CF (observed range 22–30 across five smoke runs of Classic).

Why it matters: multiple replicates per config absorb the nondeterminism into Monte Carlo error bars. Single-replicate smoke results should be read as one random draw, not a deterministic baseline. Dependency-injecting ZenGM's random source is tracked future work.

License: source-available, private research use only

The cloned kvr06-ai/zengm fork is gitignored from the public paper repo; no modified ZenGM source is pushed anywhere. Aggregate metrics (CSV, Pareto plots) are derivative work and publishable as scholarly output.

Why it matters: keeps the public-facing artifact (this page, the CSVs, the paper) cleanly separable from the in-tree engine fork.

16-tiered pool approximated by 16 worst-record teams

The 16-tiered E dial (paper's 3-2-1 proposal pool) is approximated in the driver as "the 16 worst-record teams" rather than the exact tier composition (10 non-playoff non-play-in + 4 record-9/10 + 2 7v8 losers).

Why it matters: cfg 32 (the named 3-2-1 baseline) is dominated regardless. Cfg 45 (16-tiered, C=200, unbounded) appears on the frontier and should be re-checked against the exact tier weighting before submission — tracked as a follow-up ticket.
